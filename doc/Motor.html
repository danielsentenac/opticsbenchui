Motor Configuration

Motor devices are accessible from OpticsBenchUI through multiple communication protocols (serial, Ethernet, USB), depending on the controller.

The top configuration settings box contains the information needed to establish communication with the motor controller:

Communication settings
name This is a unique name given by the user defining the communication instance.
type The communication type:
  • ETHERNET
  • SERIAL_PORT
  • USB_COM
address The address settings:
  • ETHERNET: controller IP address
  • SERIAL_PORT: serial device (example: /dev/ttyS0)
  • USB_COM: device path if required by the driver
settings Additional settings, comma-separated:
  • ETHERNET: port, read delay (s), write delay (s)
  • SERIAL_PORT: baud rate, data bits, parity, stop bits, vtime, flow control
  • USB_COM: vendor/product IDs, e.g. [VENDOR=0x3024,PRODUCT=0x1000]

The second configuration settings box defines the motor driver type, which depends on the controller used:

Driver settings
name This is a unique name given by the user defining the driver instance.
com The communication instance used for the present driver (the communication instance name defined in the Communication settings box).
type The driver type (examples):
  • NewFocus_8750_Ol: New Focus Ethernet Picomotor 8750 (open loop)
  • NewFocus_8750_Cl: New Focus Ethernet Picomotor 8750 (closed loop)
  • Pi_C509: PI C-509 position servo controller
  • Pi_C862: PI C-862 Mercury DC motor controller
  • Pi_E725: PI E-725 controller
  • NewPort_NSC200: Newport NSC200 controller (optionally with NSC-SB switchbox)
  • NewPort_AGUC2: Newport AGUC2 controller
  • Micos_Pollux: Micos SMC Pollux controller
  • Standa_uSMC: Standa uSMC controller (legacy)
  • Standa_uSMC2: Standa uSMC2 controller (XIMC)
settings Additional settings, in format [key=value] separated by spaces:
  • NewFocus_8750_Ol: 1
  • NewFocus_8750_Cl: 1
  • Pi_C509: NONE
  • Pi_C862: deviceNumber=1
  • Pi_E725: (driver-specific)
  • NewPort_NSC200: motor=0 type=0 frequency=100
  • NewPort_AGUC2: (driver-specific)
  • Micos_Pollux: axisNumber=1
  • Standa_uSMC: (driver-specific)
  • Standa_uSMC2: (driver-specific)

The third configuration settings box defines actuators (axes) driven by the selected driver:

Actuator settings
name This is a unique name given by the user defining the actuator instance.
driver The driver instance used for the present actuator (the driver instance name defined in the Driver settings box).
description A brief description of the actuator.
settings Additional settings, in format [key=value] separated by spaces:
  • NewFocus_8750_Ol: motor=0 type=0 frequency=100
  • NewFocus_8750_Cl: motor=0 type=0 frequency=100
  • Pi_C509: A
  • Pi_C862: pidP=250 pidI=12 pidD=250 integralLimit=2000 acc=900000 vel=150000
  • NewPort_NSC200: channel=1 vel=150
  • Pi_E725: axisNumber=1 pause=100
  • NewPort_AGUC2: axisNumber=1
  • Standa_uSMC: axisNumber=1 rotator=1 speed=200 divisor=8
  • Standa_uSMC2: axisNumber=1
  • Micos_Pollux: vel=0.5
