Motor Configuration

Motor devices are accessible from OpticsBenchUI through various possible communication protocols (serial, ethernet, usb).

The first top configuration settings box deals with the necessary information to establish a communication with the motor controller:

Communication settings
name This is a unique name given by the user defining the communication instance
type The communication type:
  • ETHERNET
  • SERIAL_PORT
  • USB_COM
    		•
    address The address settings:
  • ETHERNET: the controller IP address
  • SERIAL_PORT: The serial com port (ex: /dev/ttyS0)
  • USB_COM:
    • settings Additional settings communication type specific separated by commas:
  • ETHERNET: The communication port, the read delay in seconds, the write delay in seconds
  • SERIAL_PORT: The baud rate, the read delay in tenths of second, the software flow control (XONXOFF, default none) or hardware flow control (RTSCTS, default none)
  • USB_COM: The vendor ID and the product ID in the form [VENDOR=0x3024,PRODUCT=0x1000]
    		•

    The second configuration settings box deals with the motor driver types which is closely related to the controller used:

    Driver settings
    name This is a unique name given by the user defining the driver instance
    com The communication instance used for the present driver (the communication instance name defined in the Communication settings box)
    type The driver type:
  • NewFocus_8750_Ol: New Focus Ethernet Picomotor 8750 driver for open loop motors
  • NewFocus_8750_Cl: New Focus Ethernet Picomotor 8750 driver for closed loop motors
  • Pi_E509: PI E-509 position servo-control driver
  • Pi_C862: PI C-862 Mercury DC-Motor controller driver
  • Pi_E725: PI E-725 Controller driver
  • NewPort_NSC200: NewPort NSC200 motor controller driver alone or in combination with NSC-SB NewStep 8-axis switchbox
  • NewPort_AGUC2: NewPort AGUC2 motor controller driver
  • Micos_Pollux: Micos SMC Pollux motor controller driver
  • Standa_uSMC: Standa motor controller driver
  • Trinamic_SixPack: Trinamic SixPack2 6-Axis stepper motor controller driver
    		•
    settings Additional settings at the level of actuators usually in format : [key=value] separated by one space:
  • NewFocus_8750_Ol: 1
  • NewFocus_8750_Cl: 1
  • Pi_E509: NONE
  • Pi_C862: deviceNumber=1
  • Pi_E725:
  • NewPort_NSC200: motor=0 type=0 frequency=100
  • NewPort_AGUC2:
  • Micos_Pollux: axisNumber=1
  • Standa_uSMC:
  • Trinamic_SixPack: address=0 vel=100
    		•

    The third configuration settings box deals with the actuator types which is closely related to the motor used:

    Actuator settings
    name This is a unique name given by the user defining the actuator instance
    driver The driver instance used for the present actuator (the driver instance name defined in the Driver settings box)
    description A brief description of the actuator
    settings Additional settings at the level of actuators usually in format : [key=value] separated by one space:
  • NewFocus_8750_Ol: motor=0 type=0 frequency=100
  • NewFocus_8750_Cl: motor=0 type=0 frequency=100
  • Pi_E509: A
  • Pi_C862: pidP=250 pidI=12 pidD=250 integralLimit=2000 acc=900000 vel=150000
  • NewPort_NSC200: channel=1 vel=150
  • Pi_E725: axisNumber=1 pause=100
  • NewPort_AGUC2: axisNumber=1
  • Standa_uSMC: axisNumber=1 rotator=1 speed=200 divisor=8
  • Micos_Pollux: vel=0.5
  • Trinamic_SixPack: motor=1 pimode=0 axismode=1 fullsteps=100000
    		•
