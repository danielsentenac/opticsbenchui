Acquisition Configuration

The Acquisition window allows the user to program a scan and save data in a user-defined way.

Acquisition sequence
record This is a unique value given by the user defining the sequence order in the scan. The scan is processed in ascending order.
type The type of sequence. Supported values include:
  • MOTOR: a motor is called for a movement
  • SUPERK: a supercontinuum laser is called for filter settings
  • DAC: a DAC is called for setting an output value
  • CAMERA: a camera is called for taking an image
  • TREATMENT: a calculation on existing sequence data is performed
  • FILE: an external file is parsed to use its data
  • SLM: a spatial light modulator (secondary screen) is used with a registered mask list
instrument The instrument instance name from its configuration. Cameras are numbered from 0 to n in order of appearance in the Instrument menu.
settings Instrument actions, usually in format [key=value] separated by spaces.
  • MOTOR: MOVEREL (relative movement) or MOVEABS (absolute movement) with the target value
  • SUPERK: (R)POWER (power level), (R)ND (output level), (R)SWP (short wavelength setpoint), (R)LWP (long wavelength setpoint) with absolute or relative values
  • DAC: OUTPUT key with VALUE (absolute) or RVALUE (relative) for the output value
  • CAMERA: SNAPSHOT for Mono 8-bit; SNAPSHOT32 for Mono 12-bit, Mono 12-bit packed, or Mono 16-bit
  • TREATMENT: AVG=[i[-+/*]j] performs arithmetic between two sequence records i and j (same instrument type). Example: AVG[10] uses the average value from record 10. PHASE=[i,j] computes phase from imaginary i and real j. AMPLITUDE=[i,j] computes amplitude from imaginary i and real j. IMAGE=[i] converts a 2D array dataset to an 8-bit pixel image.
  • FILE: see File section.
  • SLM: see SLM section.
  • ANALYSIS=START: trigger the Analysis task list
  • SLEEP: delay (seconds) added at the end of the sequence
scanplan Scan and data-saving behavior, usually in format [key=value] separated by spaces.
  • LOOP: a fixed number of loops begin when this key is present and must be terminated with LOOP=END. More than one loop can be nested.
  • LOOPMODE: defines loop traversal behavior. Use LOOPMODE=SNAKE for a serpentine scan (alternate direction each loop pass).
  • DATAGROUP: assign a name to a group of data, often used with LOOP.
  • DATANAME: assign a name to a data element. Scalar data is attached to the parent group as an attribute. Averaged data (defined in the settings box) is attached to the grandparent group.
