OpticsBenchUI Documentation (version 1.6)

OpticsBenchUI is dedicated to automating optics lab experiments. It integrates open-source projects such as libdc1394, Aravis, and Comedi to coordinate cameras, motion stages, and I/O hardware from a single interface. Each instrument can be configured and operated independently, while complex, multi-instrument scan strategies are programmed in the Acquisition panel.

Supported devices include digital cameras (FireWire/DCAM, GigE Vision, Camera Link), actuators (RS-232, USB, Ethernet), spatial light modulators (SLM), digital and analog I/O (counters, DAC, ADC), and Supercontinuum laser sources (CP210X USB-UART bridge). Data are stored in a hierarchical, scan-aware layout using HDF5. The graphical user interface is built with Qt.

Motivation

Optics bench experiments typically require software for two complementary tasks:

  1. Individual device control and acquisition (motors, cameras, shutters, DAC/ADC, lasers) for manual alignment and setup.
  2. Coordinated, automated workflows where multiple devices must interact during scans or control loops.

Commercial frameworks can address the second point, but often become difficult to extend as experiments grow in complexity. An open-source approach offers long-term flexibility, transparent behavior, and easier reuse of software components.

OpticsBenchUI embraces open-source principles: it is modifiable, hardware-agnostic, and benefits from a broad ecosystem of libraries and vendor SDKs. This makes it a practical bridge between academic research and industrial workflows.

  • Comedi devices for acquisition and control (Configuration)
  • DAC boards to set voltages/currents (Configuration)
  • Supercontinuum sources (SuperK) (Configuration)
  • Motors to move optics (Configuration)
  • Cameras for beam profiling and imaging (Configuration)
  • SLMs to modulate phase and amplitude (Configuration)
  • File inputs for external data (Configuration)
  • Analysis tasks (external commands) (Configuration)

Once devices are configured, scans are run from the Acquisition panel in the Operation menu (Configuration).

Data are saved in HDF5 files (accessible from C/Fortran and many other languages). See the HDF5 documentation.

Build-time options and compilation flags are documented in Compilation Options.

External Libraries (.pro)

The OpticsBenchUI.pro file wires optional third-party SDKs/libraries through build defines and include/library paths. If a dependency is missing, the related feature is disabled at build time.

Data and Plotting (HDF5 / NO_HDF5, COMEDICOUNTER, COMEDIDAC)

  • HDF5 data files (HDF5 / NO_HDF5): HDF5
  • Comedi drivers/libraries (COMEDICOUNTER, COMEDIDAC / NO_COMEDI): Comedi
  • Qwt plotting (COMEDICOUNTER): Qwt

Cameras (USBCAMERA / NO_MULTIMEDIA, IEEE1394CAMERA, GIGECAMERA, NEOCAMERA, ZYLACAMERA, RAPTORFALCONCAMERA, RAPTORNINOX640CAMERA, RASPICAMERA, ALLIEDVISIONCAMERA, NO_USB)

  • USB access (NO_USB / USB comm): libusb
  • IEEE1394/DCAM (IEEE1394CAMERA / NO_DC1394): libdc1394 and libraw1394
  • GigE Vision (GIGECAMERA / NO_ARAVIS): Aravis
  • Andor Neo/Zyla (NEOCAMERA, ZYLACAMERA / NO_NEO): Andor SDK3 (atcore)
  • Allied Vision cameras (ALLIEDVISIONCAMERA): Vimba X SDK
  • Raspberry Pi camera (RASPICAMERA / NO_RASPI): raspicam
  • Raptor cameras (RAPTORFALCONCAMERA, RAPTORNINOX640CAMERA / NO_RAPTOR): Raptor support and EPIX XCLIB

Motors (Standa when available)

  • Standa motors (NO_STANDA): XIMC (libximc)

DACs and SPI (ADVANTECHDAC / NO_ADVDAQ, RASPIDAC / NO_SPI)

  • Advantech DACs (ADVANTECHDAC / NO_ADVDAQ): DAQNavi SDK
  • Raspberry Pi SPI (RASPIDAC / NO_SPI): bcm2835

Compilation Options (OpticsBenchUI.pro DEFINES)

Build flags are defined in OpticsBenchUI.pro. Missing SDKs automatically disable their features.

Core data/plotting

  • HDF5 / NO_HDF5: Enable/disable HDF5 storage.
  • COMEDICOUNTER: Enable Comedi counter support (requires Qwt).
  • COMEDIDAC: Enable Comedi DAC support.
  • NO_COMEDI: Disable all Comedi support.

Cameras

  • USBCAMERA / NO_MULTIMEDIA: Qt Multimedia camera backend.
  • IEEE1394CAMERA / NO_DC1394 / NO_RAW1394: FireWire cameras via libdc1394/libraw1394.
  • GIGECAMERA / NO_ARAVIS: Aravis GenICam GigE/USB3 cameras.
  • NEOCAMERA / ZYLACAMERA / NO_NEO: Andor SDK3 cameras.
  • ALLIEDVISIONCAMERA: Allied Vision Vimba X SDK.
  • RASPICAMERA / NO_RASPI: Raspberry Pi raspicam support.
  • RAPTORFALCONCAMERA / RAPTORNINOX640CAMERA / NO_RAPTOR: Raptor/EPIX cameras.
  • NO_USB: Disable USB comms (libusb).

Motion / motors

  • NO_STANDA: Disable Standa XIMC motors.

DAC / IO

  • ADVANTECHDAC / NO_ADVDAQ: Advantech DAQNavi SDK.
  • RASPIDAC / NO_SPI: Raspberry Pi SPI DAC support.
