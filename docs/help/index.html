OpticsBenchUI Documentation (version 1.1)

OpticsBenchUI controls devices commonly used in optical experiments. The exact hardware support depends on build options and installed drivers.

  • Comedi devices for acquisition and control (Configuration)
  • DAC boards to set voltages/currents (Configuration)
  • Supercontinuum sources (SuperK) (Configuration)
  • Motors to move optics (Configuration)
  • Cameras for beam profiling and imaging (Configuration)
  • SLMs to modulate phase and amplitude (Configuration)
  • File inputs for external data (Configuration)

Once devices are configured, scans are run from the Acquisition panel in the Operation menu (Configuration).

Data are saved in HDF5 files (accessible from C/Fortran and many other languages). See the HDF5 documentation.

External Libraries (.pro)

The OpticsBenchUI.pro file wires optional third-party SDKs/libraries through build defines and include/library paths. If a dependency is missing, the related feature is disabled at build time.

Data and Plotting (HDF5 / NO_HDF5, COMEDICOUNTER, COMEDIDAC)

  • HDF5 data files (HDF5 / NO_HDF5): HDF5
  • Comedi drivers/libraries (COMEDICOUNTER, COMEDIDAC / NO_COMEDI): Comedi
  • Qwt plotting (COMEDICOUNTER): Qwt

Cameras (USBCAMERA / NO_MULTIMEDIA, IEEE1394CAMERA, GIGECAMERA, NEOCAMERA, ZYLACAMERA, RAPTORFALCONCAMERA, RAPTORNINOX640CAMERA, RASPICAMERA, ALLIEDVISIONCAMERA, NO_USB)

  • USB access (NO_USB / USB comm): libusb
  • IEEE1394/DCAM (IEEE1394CAMERA / NO_DC1394): libdc1394 and libraw1394
  • GigE Vision (GIGECAMERA / NO_ARAVIS): Aravis
  • Andor Neo/Zyla (NEOCAMERA, ZYLACAMERA / NO_NEO): Andor SDK3 (atcore)
  • Allied Vision cameras (ALLIEDVISIONCAMERA): Vimba X SDK
  • Raspberry Pi camera (RASPICAMERA / NO_RASPI): raspicam
  • Raptor cameras (RAPTORFALCONCAMERA, RAPTORNINOX640CAMERA / NO_RAPTOR): Raptor support and EPIX XCLIB

Motors (Standa when available)

  • Standa motors (NO_STANDA): XIMC (libximc)

DACs and SPI (ADVANTECHDAC / NO_ADVDAQ, RASPIDAC / NO_SPI)

  • Advantech DACs (ADVANTECHDAC / NO_ADVDAQ): DAQNavi SDK
  • Raspberry Pi SPI (RASPIDAC / NO_SPI): bcm2835

Hardware Protocols (by Device Type)

These tables summarize which communication protocols are used by the supported hardware families and which vendor SDK or system library is needed at build/run time.

Cameras

Hardware Protocol SDK/Library Notes
Generic USB cameras (UVC) USB Qt Multimedia Uses OS camera backend via Qt Multimedia.
IEEE1394 / DCAM cameras IEEE 1394 (FireWire) libdc1394 + libraw1394 Requires FireWire host/controller.
GigE Vision cameras GigE Vision (Ethernet) Aravis GenICam stack for GigE/USB3 cameras.
Andor Zyla (sCMOS) USB 3.0 or Camera Link Andor SDK3 Vendor SDK required.
Andor Neo (sCMOS) Camera Link (3-tap) Andor SDK3 Requires Camera Link frame grabber.
Allied Vision cameras GigE Vision or USB 3.0 Vimba X SDK Transport layer depends on camera model.
Raptor cameras (Falcon / Ninox) Camera Link (frame grabber) EPIX XCLIB Requires frame grabber + XCLIB.
Raspberry Pi Camera CSI-2 (MIPI) raspicam Requires Raspberry Pi CSI-2 connector.
